## How to think about workflows

<Tip>
  **Start with one workflow.** Don't try to use everything at once. Pick your most painful daily task — usually email — and let Fluso handle that for a week. Then add the next workflow.
</Tip>

<Tip>
  **Compounding integrations matter.** Gmail + Calendar + Slack + GitHub together is far more powerful than any single integration. Fluso can do things like *"When a client emails about a bug, find the related GitHub issue and draft a status update."*
</Tip>

<Tip>
  **The first week is an investment.** Fluso learns your contacts, projects, and patterns. By week 2 it's significantly more helpful than week 1.
</Tip>
